## 日常の診療フロー

```
① 受付スタッフ
   reception.html を開く
   → 診察券番号・生年月日・受診日を入力
   → 「QRコードを発行する」ボタン
   → QRコードを患者さんのスマホで読み取ってもらう

② 患者さん（待合室・来院前）
   スマホでQRをスキャン
   → 症状スコア・POEMスコア・薬の残量を入力
   → 「送信します」をタップ

③ 医師（診察中）
   カルテダッシュボードを開く（iPad）
   → 患者の回答・POEM・薬残量を確認
   → 診察後に「医師コメント」「次回受診日」を入力して保存

④ 医師（処方決定後）
   atopic_calculator.html を開く
   → 診察券番号・受診日・薬剤・塗布部位・頻度を入力
   → 計算結果を確認
   → ヘッダーの「💾 保存」ボタン → VisitHistory に記録
   → 次回来院時、患者フォームに「前回の処方」が自動表示される
```

## 各画面の説明

### ① reception.html（受付専用）

受付スタッフが使用するQRコード発行画面です。

**入力項目：**
- 診察券番号（例：`09539`）
- 生年月日（年齢の自動計算に使用）
- 受診日（自動で今日の日付が入る）

**発行後の表示：**
- QRコード
- 診察券番号・受診日時点の年齢
- 有効期限（7日間）

> 患者さんの名前は保存しません（プライバシー保護のため）。

---

### ② atopic_calculator.html（医師用処方計算）

医師が処方量を計算する画面です。

**主な機能：**
- 受診日・次回受診日・経過日数の計算
- 薬剤ごとのFTU・必要量・処方量の計算
- テキスト出力（電子カルテへのコピー用）
- 患者向け共有（塗り方QR・印刷）
- **「💾 保存」ボタン** → 処方内容をGoogle Sheetsに保存

---

### ③ カルテダッシュボード（医師用）

患者が送信したアンケートを確認・コメントする画面です。

**表示内容（1件ごと）：**
- 診察券番号・年齢
- 症状スコア（なし〜最重度）
- POEMスコア（7項目・合計点・重症度）
- 薬の残量（各薬ごと）
- 前回の処方内容

**操作：**
1. 未確認タブを開く
2. 内容を確認
3. 「医師コメント」を入力（処方方針・メモ）
4. 「次回受診日」を選択
5. 「保存して確認済みにする」→ 確認済みタブに移動

## スプレッドシートの構成

### PatientRegistry（患者台帳）

| 列 | ヘッダー | 内容 | 備考 |
|---|---|---|---|
| A | `patientNo` | 診察券番号 | 書式：テキスト（先頭ゼロに注意） |
| B | `birthdate` | 生年月日 | 例：`2020-05-15`（reception.htmlから自動入力） |
| C | `notes` | 備考 | アレルギー情報など（任意） |
| D〜G | トークン情報 | 自動設定 | 手入力不要 |

> **D〜G列は触らないでください。** QR発行時に自動で書き込まれます。

### VisitHistory（処方履歴）

| 列 | ヘッダー | 内容 |
|---|---|---|
| A | `patientNo` | 診察券番号 |
| B | `visitDate` | 受診日（例：`2026-05-19`） |
| C | `nextVisitDate` | 次回受診予定日 |
| D | `drugsJson` | 処方薬（JSON・自動） |
| E | `rxSummaryText` | 処方サマリー（自動） |

> 「💾 保存」ボタンを押すと自動で記録されます。手入力不要。

## 患者の新規登録方法

### 初診患者（当日来院）

1. `reception.html` を開く
2. 診察券番号・生年月日・受診日を入力
3. 「QRコードを発行する」→ 患者さんに渡す
4. PatientRegistry シートに自動で行が追加される

### 既存患者の一括登録（移行時）

PatientRegistry シートに直接入力します。

1. A列（patientNo）のセル書式を**「書式なしテキスト」**に設定
2. A列に診察券番号、B列に生年月日（`YYYY-MM-DD`形式）を入力
3. C列に備考があれば入力

## よくある操作

| やりたいこと | 操作 |
|---|---|
| 新患のQRを発行 | `reception.html` → 番号・生年月日入力 → 発行 |
| 再診のQRを発行 | `reception.html` → 同じ番号・生年月日入力 → 発行（トークンが更新される） |
| 処方を計算・保存 | `atopic_calculator.html` → 計算後「💾 保存」 |
| 患者の回答を確認 | カルテダッシュボード → 未確認タブ |
| トークンを無効化 | `PatientRegistry` の該当行G列（isActive）を `FALSE` に変更 |
| 備考を追加 | `PatientRegistry` の該当行C列に入力 |

---

## 注意事項

- 診察券番号は**先頭ゼロを含む文字列**で管理してください（例：`09539`）。シートのA列は必ず「書式なしテキスト」に設定すること。
- QRコードの有効期限は**7日間**です。再来院時は毎回新しいQRを発行してください。
- `AuditLog` シートは認証ログが自動記録されます。手入力・編集しないでください。
- ダッシュボードのURLに含まれる `secret=` の値は他人に教えないでください。
