# 運用マニュアル — 患者登録・管理

## スプレッドシートの確認

GASデプロイ後、GASエディタから `setupSheets()` を一度実行すると、以下の4シートが自動作成されます。

| シート名 | 用途 |
|---|---|
| `PatientRegistry` | 患者台帳・トークン管理 |
| `VisitHistory` | 処方履歴 |
| `PatientReports` | 患者フォーム回答 |
| `AuditLog` | トークン認証ログ（自動記録） |

---

## A. 既存患者の一括登録（スプレッドシート直接入力）

既存患者は `PatientRegistry` シートに直接入力するのが最もシンプルです。

### 入力する列（A〜D列のみ手入力。E〜H列は自動設定される）

| 列 | ヘッダー | 入力内容 | 例 |
|---|---|---|---|
| A | `patientNo` | 診察券番号（文字列） | `1001` |
| B | `patientName` | 患者氏名 | `山田 花子` |
| C | `defaultAgeGroup` | 年齢グループ | `乳児`・`幼児`・`学童`・`成人` のどれか |
| D | `notes` | 備考（任意） | `アレルギー：卵`、`保護者：山田太郎` など |

> **E〜H列（tokenHash / tokenSalt / tokenExpiresAt / isActive）は空白のままで構いません。**  
> 診察時にatopy_v9.htmlから「📱 QRフォームを生成」ボタンを押すと自動で埋まります。

### 手順

1. Google Sheets を開く
2. `PatientRegistry` シートを選択
3. 2行目以降に1患者1行で入力（1行目はヘッダー行のため触らない）
4. 入力例：

```
patientNo | patientName | defaultAgeGroup | notes
1001      | 山田 花子    | 幼児            | アレルギー：卵
1002      | 鈴木 一郎    | 乳児            | 保護者：鈴木 次郎
1003      | 佐藤 あおい  | 学童            |
```

## B. 新規患者の登録

### パターン1：来院前に登録しておく（推奨）

初診患者が来院したとき、診察前に `PatientRegistry` に手入力しておくと、患者フォームに患者名が正しく表示されます。

1. `PatientRegistry` シートを開く
2. 新しい行に `patientNo`（診察券番号）・`patientName`・`defaultAgeGroup` を入力
3. atopy_v9.html を開き、その患者番号で「📱 QRフォームを生成」→ QRコードを患者に渡す

### パターン2：QR生成を先に行う（簡易）

シートへの事前入力なしで QR だけ先に発行することもできます。

1. atopy_v9.html で患者番号を入力し「📱 QRフォームを生成」を押す
2. QR発行時に `PatientRegistry` へ自動的に行が追加される（名前・年齢グループは空白）
3. 診察後、シートを開いてB列（patientName）・C列（defaultAgeGroup）を手入力で補完する

> **注意：** 名前を後から入力した場合、**次回来院時のフォームから**患者名が表示されます。初回はフォーム上の名前欄が「（未設定）」と表示されます。

## C. 処方履歴の登録（任意・推奨）

`VisitHistory` シートに処方履歴を入力すると、患者フォームに「前回の処方」が表示され、薬の残量確認もできるようになります。

### 列構成

| 列 | ヘッダー | 内容 | 例 |
|---|---|---|---|
| A | `patientNo` | 診察券番号 | `1001` |
| B | `visitDate` | 診察日 | `2026-05-01` |
| C | `nextVisitDate` | 次回受診予定日 | `2026-06-05` |
| D | `drugsJson` | 処方薬（JSON形式） | 下記参照 |
| E | `rxSummaryText` | 処方サマリー（テキスト） | `プロトピック軟膏 顔 1日2回` |

### drugsJson の書き方

```json
[
  {"name": "プロトピック軟膏0.03%", "partLabel": "顔・首", "freqLabel": "1日2回"},
  {"name": "ロコイド軟膏", "partLabel": "体幹", "freqLabel": "1日1回"},
  {"name": "ザイザルシロップ", "partLabel": "", "freqLabel": "寝る前"}
]
```

- `name`：薬品名（患者フォームに表示される）
- `partLabel`：塗る部位（任意）
- `freqLabel`：使用頻度（任意）

> **ヒント：** 最初は `rxSummaryText`（E列）に簡単なテキストを入れるだけでも構いません。`drugsJson` は残量確認ボタンの生成に使われるため、できれば入力することを推奨します。

## D. 日常の診療フロー（完成後の運用）

```
【来院前】
患者 → QRスキャン → フォーム入力 → 送信（PatientReports に自動保存）

【診察中】
医師 → 医師ダッシュボード（iPad）を開く
     → 患者の回答・POEM・薬残量を確認
     → 「医師コメント」「次回受診日」を入力して保存

【診察後】
atopy_v9.html → 処方内容を入力・確定
              → VisitHistory に処方履歴を追記（将来：自動同期ボタン予定）
```

## E. よくある操作チートシート

| やりたいこと | 操作 |
|---|---|
| 既存患者を一括登録 | `PatientRegistry` シートにA〜D列を直接入力 |
| 新規患者のQRを発行 | atopy_v9.html → 患者番号入力 → 「QRフォームを生成」 |
| 患者名を後から登録 | `PatientRegistry` の該当行B列に患者名を入力 |
| 処方履歴を登録 | `VisitHistory` に1診察1行で入力 |
| 患者フォームの回答を確認 | 医師ダッシュボード（`...exec?page=dashboard&secret=〇〇`） |
| トークンを無効化 | `PatientRegistry` の該当行H列（isActive）を `FALSE` に変更 |

---

## F. 注意事項

- `patientNo`（診察券番号）は **文字列として統一**してください。先頭ゼロがある場合（例：`0042`）はシートのセル書式を「書式なしテキスト」に設定してください。
- トークンは atopy_v9.html 上で生成・発行するたびに自動更新されます。一度発行した QR の有効期限が切れたら、再度発行してください。
- `AuditLog` シートは自動記録されます。手入力しないでください。
